# Product Shell Source-Art Preproduction

This folder contains bounded source-art preproduction for the future shell-facing layer around Apartment 302.
It is not runtime UI, not Unreal content, and not implementation-ready shell code.

The goal of this pass is to define:

- title visual direction
- lobby / board visual direction
- briefing visual direction
- debrief visual direction
- motif carry-forward from the accepted Apartment 302 pack
- palette and typography intent
- boundary notes so later shell work does not silently expand scope

## Direction summary

- Keep the shell readable first, occult second.
- Carry forward the Apartment 302 tone through restrained ash white, old gold, hospital green-gray, and paper-black values rather than through literal nursery props.
- The shell should feel like a sincere municipal-exorcism process wrapped around a haunted apartment case.
- Title and lobby can be more graphic and poster-like than in-room clue art, but briefing and debrief should stay calm, report-like, and stream-readable.

## Typography intent

- display direction: tall condensed uppercase with quiet institutional authority
- body direction: plain readable sans-serif with strong spacing and no faux-horror ornament
- accent direction: small-caps labels, stamped tokens, and restrained tab markers

Do not treat the named directions above as locked fonts.
They are style targets for later implementation or a future higher-fidelity source-art pass.

## Boundary notes

- This pack is source-art-only preproduction.
- Do not place these files into Unreal content folders.
- Do not treat these boards as finished HUD layouts.
- Do not let the shell become louder than the Apartment 302 case itself.
- Do not assume `GP-A1` or shell implementation is already active just because this preproduction pack exists.
