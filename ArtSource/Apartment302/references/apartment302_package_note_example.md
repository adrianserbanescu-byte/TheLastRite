# Apartment 302 Package Note Example

Use this as a copy-ready example when a packaged-content checkpoint or screenshot review needs one concise GitHub-visible note.

Pair it with:

- `apartment302_screenshot_note_shorthand.md`
- `apartment302_review_template.md`
- `Docs/PackageValidationNoteTemplate.md`

## Example note

```text
Apartment 302 quick note

branch: codex/content
scope: package-ready content review

shots:
- A302_Shot_01_EntryWide: reads nursery-first
- A302_Shot_03_NurseryAnchor: answer path stays linked
- A302_Shot_04_MirrorTrap: mirror tempts but stays secondary
- A302_Shot_07_EntryToAnswerLine: sightline survives dressing

labels:
- nursery clue cluster: ok
- correct nursery anchor: ok
- mirror trap: ok
- false-lead balance: ok

next pass:
- reduce window emphasis slightly if later clutter starts stealing the first read
```

## Why this shape works

- it keeps the note short enough for a package-lane checkpoint
- it reuses the stable shot ids already named in review docs
- it records one clear next pass without sounding like a blocker

## Out of scope

- gameplay prompt changes
- ritual logic changes
- interaction-range tuning
- cross-lane ownership changes
