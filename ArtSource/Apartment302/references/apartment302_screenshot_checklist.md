# Apartment 302 Screenshot Checklist

Use this checklist when a room-dressing pass needs quick visual feedback without reopening every board.

Pair it with `apartment302_import_manifest.md` so screenshots, notes, and labels all use the same vocabulary.

## Goal

Capture enough repeatable views to answer three questions quickly:

1. does the nursery evidence family read first
2. does the correct ritual anchor stay connected to that family
3. do the false leads distract without taking over the room

## Required screenshot set

Take these views in a consistent order whenever a meaningful room-dressing pass is ready for review.

| Shot id | Camera intent | What should read first | What to watch for |
| --- | --- | --- | --- |
| `A302_Shot_01_EntryWide` | first player entry sweep | nursery clue cluster | mirror trap should not outrank the nursery on first glance |
| `A302_Shot_02_NurseryCluster` | medium view of the clue family | `A302_Clue_CradleHalo`, `A302_Clue_CardCrown`, `A302_Clue_BabyMonitor` | clue spacing should feel authored, not scattered |
| `A302_Shot_03_NurseryAnchor` | medium or low view of the answer area | `A302_Anchor_NurserySigil` | correct anchor should still feel like the answer without UI help |
| `A302_Shot_04_MirrorTrap` | temptation view of the wrong anchor | `A302_Trap_MirrorCircle` | it can be dramatic, but should feel colder and less evidence-backed |
| `A302_Shot_05_WindowFalseLead` | panic-route false lead | `A302_FalseLead_WindowLatch` | mundane panic read should stay believable and peripheral |
| `A302_Shot_06_PawnFalseLead` | debt-story false lead | `A302_FalseLead_PawnPouch` | should support human stress, not ritual certainty |
| `A302_Shot_07_EntryToAnswerLine` | angle from entry toward correct answer | nursery cluster plus sigil | sightline to the true answer should survive prop dressing |

## Naming rule

If screenshots are shared in notes or pull summaries, keep the shot id exactly as written above.

That way package notes, room-dressing feedback, and future verification can all refer to the same short names.

## Review pass checklist

For each screenshot set, answer these in plain language:

1. Which object or zone reads first?
2. Does that first read point toward the nursery clue family or toward a false lead?
3. Does the correct anchor still feel related to the nursery evidence?
4. Does the mirror trap feel tempting without becoming the strongest explanation?
5. Are the two quieter false leads separated enough to avoid one noisy distraction corner?

## False-lead balance quick pass

When the room is getting denser, add one short salience-order note beside the screenshot set:

```text
salience order:
1. <first read>
2. <second read>
3. <third read>
4. <fourth read>
5. <fifth read>
```

Use the target order from `apartment302_false_lead_balance_guide.md` unless a shot-specific exception is intentional and called out in the note.

## Fast labels to reuse in notes

Use these exact labels in comments when possible:

- `nursery clue cluster`
- `correct nursery anchor`
- `mirror trap`
- `window false lead`
- `pawn false lead`
- `entry-to-answer sightline`
- `false-lead balance`

## Done when

A screenshot set is good enough for handoff when:

- `A302_Shot_01_EntryWide` reads nursery-first
- `A302_Shot_03_NurseryAnchor` still feels like the answer
- `A302_Shot_04_MirrorTrap` tempts without dominating
- both false-lead shots stay believable and secondary
- the recorded salience order still keeps the nursery answer path above the false leads
- screenshot notes use the same labels as the import manifest
