# Apartment 302 Handoff Routing Note

Use this note when the next package/content pass knows the problem category but needs the fastest pointer to the right Apartment 302 reference doc.

This is a routing helper only. It does not replace the underlying review or placement notes.

## Start here when the issue is...

| If the problem is... | Open this first | Why |
| --- | --- | --- |
| first-read clarity from entry | `apartment302_screenshot_checklist.md` | it anchors the baseline views and asks what reads first |
| clutter blocking the nursery path | `apartment302_occlusion_watchlist.md` | it protects the clue-family-to-answer route as dressing increases |
| mirror, window, or pawn beats getting too loud | `apartment302_false_lead_balance_guide.md` | it preserves the intended salience order |
| a screenshot failure needs one small next fix | `apartment302_placement_triage_note.md` | it maps the failure to the smallest lane-safe adjustment |
| a quick GitHub-visible comment is needed | `apartment302_screenshot_note_shorthand.md` | it gives stable labels and short verdict phrases |
| a fuller room-dressing note is needed | `apartment302_review_template.md` | it records shots, labels, balance, and next pass in one place |
| a concise packaged-content checkpoint is needed | `apartment302_package_note_example.md` | it shows the short example shape for a package-lane note |
| stable object names or bundle labels are drifting | `apartment302_import_manifest.md` | it keeps notes, labels, and dressing references aligned |

## Fast routing rule

If more than one issue appears at once:

1. start with the entry-first read
2. then restore clue-to-answer linkage
3. then reduce false-lead overreach
4. then clean up note wording and labels

That order keeps readability fixes ahead of paperwork.

## Out of scope

- gameplay prompt changes
- ritual logic changes
- interaction-range tuning
- cross-lane ownership changes
