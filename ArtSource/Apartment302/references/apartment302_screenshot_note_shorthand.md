# Apartment 302 Screenshot Note Shorthand

Use this note when package-lane validation comments or room-dressing reviews need quick, consistent wording instead of a full template.

Pair it with:

- `apartment302_screenshot_checklist.md`
- `apartment302_review_template.md`
- `apartment302_import_manifest.md`

## Goal

Keep short package notes, screenshot comments, and handoff replies using the same labels and verdict language.

## Fast labels

Reuse these exact labels whenever possible:

- `nursery clue cluster`
- `correct nursery anchor`
- `mirror trap`
- `window false lead`
- `pawn false lead`
- `entry-to-answer sightline`
- `false-lead balance`

## Short verdict phrases

### Good states

- `reads nursery-first`
- `answer path stays linked`
- `mirror tempts but stays secondary`
- `window beat stays peripheral`
- `pawn beat lands on approach`
- `false leads stay separated`
- `sightline survives dressing`

### Problem states

- `mirror outranks nursery`
- `window steals first read`
- `pawn reads too early`
- `false leads collapse into one corner`
- `anchor feels detached`
- `entry sightline breaks`
- `clutter muddies clue family`

## Copy-ready micro note

```text
Apartment 302 quick note

shots:
- A302_Shot_01_EntryWide: <short verdict phrase>
- A302_Shot_03_NurseryAnchor: <short verdict phrase>
- A302_Shot_04_MirrorTrap: <short verdict phrase>
- A302_Shot_07_EntryToAnswerLine: <short verdict phrase>

labels:
- nursery clue cluster: <ok / issue>
- correct nursery anchor: <ok / issue>
- mirror trap: <ok / issue>
- false-lead balance: <ok / issue>

next pass:
- <smallest next in-scope adjustment>
```

## Usage rule

Use the short verdict phrases verbatim when possible. If a custom phrase is needed, keep it close to the same meaning so later package notes stay comparable.

## Out of scope

- gameplay prompt changes
- ritual logic changes
- interaction-range tuning
- cross-lane ownership changes
