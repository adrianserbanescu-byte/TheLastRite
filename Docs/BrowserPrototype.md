# Browser Prototype Scope

## Purpose

Use the browser build to validate the game loop, not to mimic the final UE production stack.

## Current slice

- first-person room exploration
- one containment scene at a time
- evidence collection
- false leads
- diagnosis board
- ordered ritual execution
- contamination pressure
- debrief result

## Deliberate scope cuts

- no real multiplayer yet
- no voice system
- no host migration
- no production assets
- no final progression/meta layer
- no shipping-quality enemy behavior

## Why this still matters

If the browser slice proves:

- the clues are readable
- the diagnosis feels fair
- the ritual creates pressure
- contamination drives good pacing

then the Unreal port has a stable gameplay target instead of a moving design brief.
